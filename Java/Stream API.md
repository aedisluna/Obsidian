| Оператор               | Описание                                                                   |     |
| ---------------------- | -------------------------------------------------------------------------- | --- |
| `Stream<T> of()`       | Создаёт поток из набора объектов                                           |     |
| `Stream<T> generate()` | Генерирует поток по заданному правилу                                      |     |
| `Stream<T> concat()`   | Объединяет несколько потоков в один                                        |     |
| `Stream<T> filter()`   | Фильтрует данные. Оставляет данные, подходящие под описанное правило.      |     |
| `Stream<T> distinct()` | Удаляет дубликаты                                                          |     |
| `Stream<T> sorted()`   | Сортирует данные                                                           |     |
| `Stream<T> peek()`     | Выполняет определённое действие над каждым элементом                       |     |
| `Stream<T> limit(n)`   | Обрезает данные после достижения лимита                                    |     |
| `Stream<T> skip(n)`    | Пропускает первые n элементов                                              |     |
| `Stream<T> map()`      | Преобразует данные из одного типа в другой                                 |     |
| `Stream<T> flatMap()`  | Преобразует данные из одного типа в другой (немного иначе, чем просто map) |     |
|                        |                                                                            |     |

**Терминальные операторы:**

| Оператор                   | Описание                                                                 |
| -------------------------- | ------------------------------------------------------------------------ |
| `boolean anyMatch()`       | Проверяет элементы потока на совпадение заданному правилу хотя бы одного |
| `boolean allMatch()`       | Проверяет все элементы потока на совпадение заданному правилу            |
| `boolean noneMatch()`      | Проверяет все элементы потока на НЕсовпадение заданному правилу          |
| `*Optional<T> findFirst()` | Возвращает первый соответствующий правилу элемент                        |
| `*Optional<T> findAny()`   | Возвращает любой соответствующий правилу элемент                         |
| `*Optional<T> min()`       | Ищет минимальный элемент в потоке                                        |
| `*Optional<T> max()`       | Ищет максимальный элемент в потоке                                       |
| `long count()`             | Возвращает количество элементов в потоке                                 |
| `R collect()`              | Все данные из потока возвращаются в виде коллекции                       |

* В некоторых методах возвращается `Optional<T>`. Optional — служебный класс. Его задача — просто хранить в себе ссылку на объект типа T. При этом ссылка на объект внутри класса `Optional<T>` может быть и null.

Изменение: метод `getStringFromStream` теперь собирает элементы стрима в строку через пробел с помощью `Collectors.joining(" ")` (src/ru/courses/streams/Main.java).

Как решено:

- `Collectors.joining(" ") `— готовый коллектор, который соединяет все элементы стрима в одну строку, вставляя заданный разделитель.

- Вызов `stringStream.collect(...)` запускает терминальную операцию, проходится по элементам и формирует результат.

Фрагмент:


``` java

public static String getStringFromStream(Stream<String> stringStream) {

    return stringStream.collect(Collectors.joining(" "));

}
```


Краткая памятка по Stream API:

- Создание: Stream.of(...), Collection.stream(), Arrays.stream(...).

- Промежуточные операции (ленивые, возвращают Stream): `map, filter, flatMap, sorted, distinct, peek, limit, skip.`

- Терминальные операции (завершают конвейер): `collect, forEach, reduce, count, findFirst, findAny, anyMatch/allMatch/noneMatch, toArray.`

- Коллекторы (класс Collectors): `toList/toSet, toMap, joining, groupingBy, partitioningBy, summarizingInt/averagingInt, collectingAndThen.`

- Особенности: одноразовое использование стрима; операции не изменяют исходную коллекцию; порядок сохраняется для упорядоченных источников; можно использовать parallelStream() для параллельной обработки, но только при оправданной выгоде и осторожности с состоянием.
