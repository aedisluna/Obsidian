## 🎯 Основы System

> [!info] System — нельзя создать экземпляр этого класса. Его поля и методы почти все статические и используются для стандартного ввода/вывода, работы с переменными окружения, времени и завершения программы.

### Главные задачи

- Стандартный **ввод (in)**, **вывод (out)** и **вывод ошибок (err)**.
- Получение/установка системных свойств и переменных окружения.
- Работа с временем и механизмами самой JVM.

***

## ⚙️ Основные поля System

```java
System.in  // стандартный ввод (обычно клавиатура)
System.out // стандартный вывод (обычно консоль, метод println)
System.err // стандартный вывод ошибок
```

- Все поля — статические, используются напрямую.

***

## 📁 Базовые методы System

| Метод | Описание |
| :-- | :-- |
| `currentTimeMillis()` | Возвращает текущее время в миллисекундах |
| `nanoTime()` | Высокоточное время в наносекундах |
| `exit(int status)` | Завершает выполнение программы с кодом возврата |
| `gc()` | Запускает сборку мусора |
| `getProperty(key)` | Получает системное свойство по ключу |
| `getenv(name)` | Получает значение переменной окружения |
| `arraycopy(...)` | Быстро копирует элементы одного массива в другой |
| `lineSeparator()` | Символ новой строки в текущей системе |



### Пример вывода на консоль

```java
System.out.println("Привет, Java!");
System.err.println("Это сообщение об ошибке.");
```

- `println()` — вывод c переводом строки, `print()` — без перевода строки.

***

## 🌿 Частые сценарии

```java
// Получить системное свойство
String javaVersion = System.getProperty("java.version");

long timestamp = System.currentTimeMillis();

System.exit(1); // завершить с ошибкой

// Быстро скопировать массив
int[] source = {1, 2, 3};
int[] dest = new int;
System.arraycopy(source, 0, dest, 0, source.length);
```

- Некоторые методы полезны для отладки, таймеров и работы с файлами/потоками.

***

## 💡 Подводные камни и советы

- Не выводи важные ошибки через `System.out` — используй `System.err` (особенно когда перенаправляешь вывод в файл).
- Для интерактивного ввода с клавиатуры через `System.in` рекомендуют использовать класс Scanner:

```java
Scanner scanner = new Scanner(System.in);
String name = scanner.nextLine();
```

- Не используешь System.gc() без необходимости — JVM сама решает, когда собирать мусор.

***

## Быстрая шпаргалка

- Вывод: `System.out.println(...)`
- Ввод: `System.in` + `Scanner`
- Завершить: `System.exit(status)`
- Время: `System.currentTimeMillis()`
- Копирование: `System.arraycopy(src, srcPos, dest, destPos, length)`
- Свойство: `System.getProperty("property")`
- Окружение: `System.getenv("VAR")`

***

> [!tip] В IntelliJ IDEA System.out часто используют для быстрой отладки — выводишь что угодно в консоль, смотришь результат и правишь код на лету.
