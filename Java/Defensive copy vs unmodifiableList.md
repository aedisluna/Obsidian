## 🔹 Defensive Copy vs Unmodifiable List

### 🧠 Идея

Как безопасно вернуть коллекцию из класса

---

## 🔸 Defensive Copy

```java
return new ArrayList<>(grades);
```

### ✔ Плюсы

- безопасно для внутреннего состояния
    
- внешний код может менять список
    

### ❌ Минусы

- создается копия (дороже по памяти)
    

---

## 🔸 Unmodifiable List

```java
return Collections.unmodifiableList(grades);
```

### ✔ Плюсы

- нельзя изменить
    
- быстро (без копии)
    

### ❌ Минусы

- если исходный список изменится — изменится и view
    

---

## ⚖️ Сравнение

||Copy|Unmodifiable|
|---|---|---|
|Можно менять|✔|❌|
|Защита состояния|✔|✔|
|Создание объекта|Да|Нет|

---

## 💡 Для Student

Лучше использовать:  
➡️ unmodifiableList (логично: оценки нельзя менять извне)
