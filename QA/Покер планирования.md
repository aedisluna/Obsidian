[[SCRUM]]

### **Как работает Покер планирования:**

1. **Раздача карт**:
    - Каждый член команды получает набор карт с числовыми значениями представляющими последовательность чисел Фибоначчи (1, 2, 3, 5, 8, 13, 21). Для этого используется сервис по типу: poinz.app
2. **Выбор задачи**:
    - Продуктовый владелец выбирает задачу из бэклога и кратко объясняет её, описывая требования и ожидаемый результат.
3. **Обсуждение задачи**:
    - Команда обсуждает задачу, задает вопросы и уточняет детали. Это помогает всем участникам получить общее понимание объема и сложности задачи.
4. **Оценка задачи**:
    - Каждый член команды одновременно выбирает карту с числом, которое, по его мнению, соответствует сложности или объему работы для данной задачи. Карты выкладываются лицевой стороной вниз, чтобы избежать влияния мнения других участников.
5. **Раскрытие карт**:
    - Все участники одновременно переворачивают свои карты, показывая свои оценки.
6. **Обсуждение различий**:
    - Если оценки значительно различаются, участники обсуждают причины своих оценок. Те, кто дал высокие или низкие оценки, объясняют свои мысли, приводя к более глубокому пониманию задачи для всех участников.
7. **Повторное голосование**:
    - После обсуждения участники повторно оценивают задачу, снова выбирая карты. Процесс повторяется до достижения консенсуса или схожих оценок.
