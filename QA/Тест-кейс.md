[[Тестовая документация]]

Тест-кейс — это такое описание проверки работы системы, которое может выполнить любой человек из команды, будь то тестировщик, разработчик, аналитик или даже бизнес-заказчик.

Мельчайший артефакт тестирования, одна проверка

Кратко:
1. Предусловия
2. Шаги
3. Ожидаемый результат

- должен быть подробный, понятный, детальный
- общие фразы - плохо, потому что это может дать плохую проверку

Длинно:
1. ID
2. Приоритет 
3. Ссылка на требование в спецификации
4. Заголовок
5. Предусловия, в них может быть стартовая страница
6. Шаги для проверки, детально, максимально понятно
7. Ожидаемый результат, всегда из спецификации
8. Аттач (скрин, видос, файл, что угодно), опционален, любая дополнительная инфа

## статуты тест-кейсов
- passed
- failed
- blocked
- retest
- untested
