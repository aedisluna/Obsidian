[[SOAP]] [[REST API +JSON]] 

SOAP — это буквально протокол доступа к объектам, в то время как REST — это архитектура.

Единственный формат данных, с которым работает SOAP — XML, в отличие от РЕСТ’а, где кроме JSON можно также использовать и XML, и др.

В современной разработке стараются уйти от SOAP, однако очень много систем с ним работают, даже если мы создаем новую систему вполне вероятно, что придется её интегрировать с какой-либо легаси системой. Таким образом и появляются Франкенштейны на REST, где есть микросервисы, работающие по SOAP.

**Разница между XML и HTML**
XML и HTML созданы под разные задачи:
- XML создан для работы с данными: именно для передачи данных между системами
- HTML создан для отображения данных
- XML теги не предопределены, в отличие от тегов HTML: то есть разработчики сами определяют, с какими тегами необходимо работать при передаче данных с помощью XML.


**Также нужно знать про WSDL и XSD.**

WSDL (Web Services Description Language) — файл, который используется для описания веб-сервисов: описывает все методы, которые используются в веб-вервисе. Ближайший аналог — swagger документация. Определить его просто: используется в конце ?wsdl.

XSD — это язык описания структуры XML документа. Его также называют XML Schema. То есть с помощью XSD можно строить XML. Схема построения XML
