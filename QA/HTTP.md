[[Бэкэнд]]

HTTP (Hypertext Transfer Protocol) - это протокол передачи данных, который используется для обмена информацией между клиентами и серверами в сети Интернет. HTTP является базовым протоколом для передачи данных в вебе и обеспечивает взаимодействие между веб-браузерами и веб-серверами. 

Основные характеристики протокола HTTP:
1. Клиент-Серверная архитектура: HTTP основан на модели клиент-сервер, где клиент отправляет запросы, а сервер отвечает на них. Эта архитектура обеспечивает разделение ответственности между устройством, выполняющим запрос (клиентом), и устройством, предоставляющим ресурсы или услуги (сервером).
2. Без состояния (Stateless): Протокол HTTP считается без состояниям, что означает, что каждый запрос от клиента к серверу рассматривается независимо от предыдущих запросов. Каждый запрос не сохраняет состояние между ними, что делает его простым и легко масштабируемым.
3. Методы запросов (HTTP Methods): HTTP определяет различные методы запросов, такие как GET, POST, PUT, DELETE, HEAD и другие. Каждый метод выполняет определенное действие. Например, GET используется для получения ресурса, POST для отправки данных на сервер, PUT для обновления ресурса и т.д.
4. URI (Uniform Resource Identifier): Ресурсы в HTTP идентифицируются с использованием URI. Это строка символов, которая уникально определяет ресурс, например, URL (Uniform Resource Locator) является подмножеством URI и определяет местоположение ресурса в Интернете.
5. Заголовки (Headers): Заголовки HTTP предоставляют метаданные о запросе или ответе. Они содержат информацию, такую как тип контента, язык, кэширование, аутентификация и другие параметры.
6. Тело сообщения (Message Body): Некоторые запросы и ответы могут содержать тело сообщения, которое передает данные. Например, тело запроса POST может содержать данные формы, а тело ответа может содержать HTML-код веб-страницы.
7. Состояние протокола (Status Codes): Сервер в ответе передает код состояния, который указывает на успешность выполнения запроса или возникновение ошибки. Например, код 200 означает успешный ответ, а код 404 - страница не найдена.

HTTP (Hypertext Transfer Protocol) запрос представляет собой сообщение, отправляемое клиентом (например, веб-браузером) серверу с целью запроса определенных ресурсов или выполнения определенного действия. HTTP запрос состоит из нескольких компонентов, включая метод запроса, заголовки и тело запроса (в некоторых случаях).

## Структура запроса:

1. Метод запроса (Request Method): Это ключевое слово, указывающее на тип операции, которую нужно выполнить. Некоторые из общих методов включают GET (получение ресурса), POST (отправка данных для обработки на сервере), PUT (обновление ресурса), DELETE (удаление ресурса) и многие другие.
2. URI (Uniform Resource Identifier): Это строка, которая идентифицирует ресурс, который клиент хочет запросить или модифицировать.
3. Заголовки (Headers): Заголовки содержат различные метаданные о запросе, такие как тип содержимого, язык, информация о сеансе и др.
4. Тело запроса (Request Body): Этот компонент необязателен и присутствует только в запросах, которые отправляют данные, такие как POST-запросы. Тело запроса содержит данные, которые клиент отправляет на сервер.

Эти компоненты объединяются в одно HTTP сообщение, которое отправляется серверу. Пример полного HTTP запроса:


```
bashCopy code GET /index.html HTTP/1.1 Host: www.example.com User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36 Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,*/*;q=0.8
```

При https формирование запроса немного меняется:

1. Установка TCP-соединения между клиентом и сервером, используется порт 443
2. Процесс SSL/TLS (handshake): после установки соединения происходит рукопожатие ssl/tls, включающее следующие шаги
	- приветствие, в котором сервер и клиент обмениваются поддерживаемый версия и протокола, криптографическими алгоритма и и прочей инфой
	- обмен сертификатами, в котором сервер предоставляет сертификат, в котором есть открытый ключ сервера и подпись центра сертификации
	- аутентификация, клиент проверяет достоверность сертификата
3. Формирование http запроса, как до этого, но с https вместо http
4. Шифрование данных: все данные шифруются с использованием ключа, установленного в процессе рукопожатия
5. Отправка запроса по безопасном соединению
6. Обработка сервером, после чего сервер отправляет зашифрованный ответ

## Методы HTTP

Идемпотентные методы - это методы, которые либо не изменяют состояние в базе данных, либо изменяют состояние только при первом запросе. В случае повторной отправки идентичного запроса, состояние в базе данных не изменяется. Идемпотентными методами являются: GET, PUT, DELETE, HEAD и OPTIONS.
- GET: Используется для запроса содержимого ресурса. Запрос с методом GET не должен изменять состояние сервера.
- POST: Используется для отправки данных для обработки на сервере. Обычно используется для отправки данных форм, загрузки файлов и других операций, которые могут изменить состояние сервера или создать новый ресурс.
- PUT: Используется для загрузки или обновления содержимого ресурса на сервере. Если ресурс не существует, метод PUT может создать новый.
- PUT: Используется для загрузки или обновления содержимого ресурса на сервере. Если ресурс не существует, метод PUT может создать новый.
- HEAD: Аналогичен методу GET, но сервер возвращает только заголовки без тела ответа. Этот метод полезен, когда клиенту нужны только заголовки для проверки состояния ресурса.
- OPTIONS: Используется для запроса информации о возможностях сервера или характеристиках ресурса, доступных на сервере.
- PATCH: Используется для частичного обновления ресурса на сервере. Клиент отправляет только те данные, которые должны быть изменены, без необходимости отправки полного представления ресурса.
- TRACE: Используется для диагностики. Клиент может отправить запрос TRACE серверу, и сервер вернет тот же запрос, который он получил, что может быть полезным для отладки и анализа маршрутов запросов.
- CONNECT: Зарезервирован для использования с прокси-серверами. Он устанавливает туннель к серверу, определенному по ресурсу.
