[[Бэкэнд]]

Набор принципов и ограничений, обеспечивающий простоту, удобство и масштабируемость API

Особенности REST:
1. Ресурсы и URI: ресурсы - объекты или данные, к которым предоставляется доступ по URI (https://website.domain/details)
2. HTTP методы: GET, POST, PUT, DELETE используются в REST
3. Предоставление ресурсов (Representation): ресурсы могут представляться в различных форматах (JSON, XML etc), клиенты могут получать ресурсы в удобных для них форматах
4. Самоописываемые сообщения (Self-descriptive messages): в каждом запросе и ответе достаточно информации для того что бы их можно было понять без дополнительной документации
5. Состояние и безопасность (Stateless and Security): RESTful сервисы должны быть без состояния, не хранить состояние между запросов. Запрос содержит достаточно информации для его обработки а так же защищен, обиспечивая безопасность данных
6. Кэширование: REST может использовать кэширование, улучшая производительность и снижая нагрузку на сервер

## JSON 

**Java Script Object Notation** - самый популярный формат обмена данных между приложениями

JSON-объект заключается в {}, состоит из формата ключ:значение, пары делятся запятыми. ключ - всегда string в кавычках

- a string (строка, например: "brand" или ⚠️"123" — может сбить с толку, но т.к. число в кавычках, то тип данных именно строковый)
- a number (число, например: 123)
- an object (объект, например: _{ "brand": “Apple” }_ )
- an array (массив, например: [ "apple", "orange", "kiwi" ]
- a boolean (Булев, Логический тип данных, т.е. одно из 2х значение истина и ложь: либо **true**, либо **false**)
- null (нул, отсутствие какого-либо объектного значения, не путать с нулём: 0 — это конкретное значение, тип данных которого число)

```
{
  "brand": "Apple",
  "model": "iPhone 15 Pro Max",
  "isAvailable": true,
  "display": 6.69,
  "memories": [256, 512, 1024],
  "features": {
    "tripleCamera": true,
    "faceId": true,
    "touchId": false,
    "eSIM": true
  }
}
```

==При выполнении REST API запросов необходимо выставить в Headers **Content-type** application/json. Postman такой флаг в заголовках поставит автоматически при использовании JSON в body запроса.==
