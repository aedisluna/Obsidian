# SOAP - альтернатива REST

SOAP (Simple Object Access Protocol) - это протокол обмена структурированными информационными сообщениями в веб-службах. Он предоставляет стандартный способ взаимодействия между программами, работающими на различных платформах, и может использоваться для обмена информацией в распределенных и сетевых средах.

Ключевые характеристики:
1. XML-формат сообщений: обеспечивает структурированный и расширяемый формат представления информации
2. Простота: сообщения состоят из легко понимаемых блоков XML- элементов. представляющий данные и действия к выполнению
3. Стандартные элементы и сообщения: Сообщения SOAP могут содержать стандартные элементы, такие как `<Envelope>, <Header>, и <Body>. <Envelope>` является обязательным корневым элементом и оборачивает все остальные элементы. `<Header> и <Body>` предоставляют контейнеры для мета-информации и основного содержимого сообщения соответственно.
4. Протокол независимости: SOAP может использоваться поверх различных протоколов транспорта, таких как HTTP, SMTP, TCP и другие. Он часто используется в сочетании с протоколом HTTP для обеспечения взаимодействия между клиентом и сервером через интернет.


```
<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope" xmlns:web="http://www.example.com/webservice">
    <soap:Header>
        <!-- Заголовок (опционально) -->
    </soap:Header>
    <soap:Body>
        <!-- Основное содержимое сообщения -->
        <web:GetData>
            <web:ID>123</web:ID>
        </web:GetData>
    </soap:Body>
</soap:Envelope>

```

Основан на стандартах: SOAP предоставляет стандартные соглашения и спецификации для взаимодействия между веб-службами, такие как WSDL (Web Services Description Language) для описания интерфейсов и UDDI (Universal Description, Discovery, and Integration) для регистрации и поиска веб-служб.


**Soap UI**: инструмент для тестирования веб-служб (web services) с использованием протокола SOAP

Его возможности:
1. Создание проектов: позволяет создавать проекты для тестирования веб-служб. Можно определить тест-кейсы, запросы и автоматизацию тестов
2. Импорт WSDL: SoapUI поддерживает импорт WSDL (Web Services Description Language) файлов, которые описывают структуру веб-службы. Это позволяет автоматически создавать запросы и тесты на основе описания веб-службы.
3. Создание запросов: позволяет легко создавать SOAP-запросы и выполнять их для анализа ответов
4. Тестирование различных аспектов веб-служб: функциональное тестирование, тестирование производительности, тестирование безопасности и другие
5. Автоматизация тестов: позволяет создавать и запускать автотесты

## Postman 

Инструмент для тестирования API, в том числе и с SOAP

1. Создать запрос в Postman
	- Создать запрос
	- Указать метод запроса
	- Ввести URL
	- Выбрать вкладку body и формат raw
	- Ввести тело запроса
2. Настроить заголовок Например, установите Content-Type в text/xml или application/soap+xml.
3. Отправить запрос
4. Изучить ответ
5. Протестировать различные сценарии
