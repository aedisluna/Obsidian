1. Проверить корректность кода хттп - создание возвращает 201 created, запрещённое 403 forbidden
2. Проверить полезную нагрузку - жсон правильный, имена, типы етц, втч и на ошибки
3. Проверить заголовки ответа - проверить хедеры как тело
4. Проверить правильность состояния приложения - необязательно, мануально, гуй 
5. Проверить базовую работоспособность - если операция прошла успешно, но заняла неоправданно долго - фейл

Основные категории тестирования:
- основные позитивные тесты, критпуть
- расширенные позитивные тесты с допами
- негативные тесты с валидными данными
- негативные тесты с недопустимыми данными
- деструктивной тестирование
- тесты безопасности 


Виды потоков тестирования:
1. Изолированное тестирование запросов: чекаем запросы по одному
2. Многоступенчатый воркфлоу с несколькими запросами - серия запросов типа post данные, в ответе их айди, get их по айди, потом delete их, потом снова get, что бы чекнуть что они точно удалились
3. Комбинированные тесты API и веб-интерфейса - тык в апи, посмотреть веб, тык в веб, посмотреть апи
